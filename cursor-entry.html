Esta página serve como ponto de entrada estruturado para indexação pelo Cursor.ai, fornecendo uma visão geral organizada de todos os padrões de programação de jogos documentados neste site.

Introdução

Esta documentação aborda padrões de programação específicos para desenvolvimento de jogos, baseados no livro "Game Programming Patterns" de Robert Nystrom. Os padrões são organizados em categorias que refletem diferentes aspectos do desenvolvimento de jogos.

  • Arquitetura e Performance em Games - Visão geral sobre como a arquitetura de software impacta a performance em jogos.
  • Sobre o Livro - Informações sobre o livro original que inspirou esta documentação.

Categorias de Padrões

Padrões de Design

Command Pattern

Encapsula uma solicitação como um objeto, permitindo parametrizar clientes com diferentes solicitações.

Ver documentação completa

Flyweight Pattern

Usa compartilhamento para suportar grandes quantidades de objetos de granularidade fina de forma eficiente.

Ver documentação completa

Observer Pattern

Define uma dependência um-para-muitos entre objetos para que quando um objeto mude de estado, todos seus dependentes sejam notificados.

Ver documentação completa

Prototype Pattern

Cria novos objetos clonando um objeto existente, conhecido como protótipo.

Ver documentação completa

Singleton Pattern

Garante que uma classe tenha apenas uma instância e fornece um ponto global de acesso a ela.

Ver documentação completa

State Pattern

Permite que um objeto altere seu comportamento quando seu estado interno muda.

Ver documentação completa

Padrões de Sequenciamento

Double Buffer Pattern

Evita artefatos visuais ao renderizar frames completos em um buffer secundário antes de exibi-los.

Ver documentação completa

Game Loop Pattern

Gerencia o fluxo de tempo e processamento em um jogo, independente da entrada do usuário.

Ver documentação completa

Update Method Pattern

Simula comportamentos independentes para objetos de jogo chamando um método de atualização em cada objeto a cada frame.

Ver documentação completa

Padrões Comportamentais

Bytecode Pattern

Define uma linguagem virtual para comportamentos de jogo que podem ser compilados para bytecode e interpretados.

Ver documentação completa

Subclass Sandbox Pattern

Define um conjunto de operações na classe base que as subclasses podem usar para implementar comportamentos.

Ver documentação completa

Type Object Pattern

Permite criar novos "tipos" de objetos em tempo de execução definindo comportamentos em objetos separados.

Ver documentação completa

Padrões de Desacoplamento

Component Pattern

Permite compor objetos de jogo a partir de componentes reutilizáveis em vez de herança profunda.

Ver documentação completa

Event Queue Pattern

Desacopla o remetente de uma notificação do seu receptor armazenando a mensagem em uma fila.

Ver documentação completa

Service Locator Pattern

Fornece um ponto global de acesso a um serviço sem acoplar clientes à implementação concreta.

Ver documentação completa

Padrões de Otimização

Data Locality Pattern

Organiza dados para tirar proveito do cache da CPU e melhorar a performance.

Ver documentação completa

Dirty Flag Pattern

Evita cálculos desnecessários rastreando quais entidades foram modificadas.

Ver documentação completa

Object Pool Pattern

Melhora a performance e o uso de memória reutilizando objetos em vez de criar novos.

Ver documentação completa

Spatial Partition Pattern

Otimiza consultas espaciais dividindo o espaço em regiões para reduzir o número de pares de objetos a verificar.
